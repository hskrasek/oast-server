Slack's Web API is famously RPC-in-a-trench-coat — method names like chat.postMessage, IDs in query strings, state transitions as bespoke verbs. We pointed the Council at it because it's a design that's defensible on purpose: Slack chose this shape. Watch the confidence column: where all three models converged you'll see consensus, and where only one model caught something you'll see it honestly labeled as a lone flag instead of laundered into false agreement.
